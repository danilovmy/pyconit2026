# ocr-reader

## Назначение

Скилл для работы с изображениями:
- **вход:** файл или несколько файлов изображений в `workspace/media/inbound/<имя_файла>.jpg|png|jpeg` 
- **действие:** перенос файла в `workspace/media/archive/bills`, расшифровка через DocTr, сохранение текста рядом с картинкой `<имя_файла>.txt`;
- **выход:** текст с картинки в `workspace/media/archive/<имя_файла>.txt` и callback=сообщение в указанную сессию агенту.

## Скрипт

Основной скрипт: `%USERPROFILE%/.openclaw/skills/ocr-reader/ocr.py`.

Поведение скрипта:
1. Принимает один или несколько позиционных аргументов, каждвй это **полный путь** к входному файлу изображения (обычно из `workspace_root/media/inbound`). Запускает в цикле обработку файлов по одному.
2. Проверяет, что обрабатываемый файл существуют.
3. Отправляет сервисное сообщение в указанную сессию агента через `openclaw agent send`:
   ```
   [ocr-start] file: <имя_файла>.<ext>
   ```
4. Ищет папку `media` в workspace root и создаёт при необходимости папку `media/archive/bills`.
5. Переносит файл из `путь\к\<имя_файла>.<ext>` в `workspace/media/archive/bills/<имя_файла>.<ext>`.
6. Трансформирует картинку, убирая искажения и перекосы.
7. Запускает процесс распознавания (через `DocTr.models.ocr_predictor`) на файл в `workspace/media/archive/bills/<имя_файла>.<ext>`.
8. Формирует текст файла, с построчно распознаным текстом.
9. Сохраняет результат в `workspace/media/archive/bills/<имя_файла>.txt` (UTF-8).
10. Отправляет в сессию агента сообщение через `openclaw agent send`:
   ```
   [ocr-done] file: <имя_файла>.<ext>
   ```

## Поведение агента, как вызывает скрипт и обрабатывает результат:
Когда пользователь говорит, что надо распознать одну или несколько картинок:

### Обязательный режим запуска
Агент **всегда**, при **каждом** вызове запускает OCR:
- с указанием **полный абсолютный путь к к скрипту `ocr.py`**;
- с ключом **`-a <имя_агента>`**;
- с ключом **`-sid <session_id>`**;
- с указанием **полного абсолютного пути к файлу каждой картинки**;
- **только асинхронно** через background exec;
- с `timeout` **300 секунд**.

Базовый запуск:
```bash
python <абсолютный_путь_к_ocr.py> -a <имя_агента> -sid <session_id> <абсолютный_путь_к_файлу>
```

**Не запускать несколько параллельных pipeline процессов**, если обрабатывается несколько файлов картинок!
Скрипт должен запускать **один процесс** для обработки **несколько файлов катинок одной командой**.
Абсолютные пути к картинкам передаются в скрипт списком, как позиционные аргументы. 

Пакетный запуск:
```bash
python <абсолютный_путь_к_ocr.py> -a <имя_агента> -sid <session_id> <абсолютный_путь_к_файлу_1> <абсолютный_путь_к_файлу_2> <абсолютный_путь_к_файлу_3>
```

Если входной файл **уже лежит в** `workspace/media/archive/bills` или не должен быть перемещён, запускать с флагом **--keep-on-place**:
```bash
python <абсолютный_путь_к_ocr.py> -a <имя_агента> -sid <session_id> --keep-on-place <абсолютный_путь_к_файлу>
```

### Подтверждение старта процесса и окончания процесса 
   агент **не имеет права** писать пользователю, что OCR «запущен», до прихода **сервисного сообщения**, подтверждающего старт: `[ocr-start] file: <имя файла>.<ext>`.

   После получения служебного сигнала как **единственного допустимого признака старта** агент ДОЛЖЕН **сообщить о старте в оба канала**:
   - в текущий чат
   - в Telegram

   Агент **не имеет права** писать пользователю, что OCR «завершён», до прихода **сервисного сообщения**, подтверждающего окончание процесса: `[ocr-done] file: <имя файла>.<ext>`.
   После получения служебного сигнала как **единственного допустимого признака оеончания процесса** агент ДОЛЖЕН **сообщить об окончании в оба канала**:
   - в текущий чат
   - в Telegram

### Если OCR не завершен, завершен с ошибкой, завершен без `.txt`:
   1. Агент должен сообщить об ошибке в оба канала:
      - в текущий чат
      - в Telegram
   2. Перезапустить процесс с ключом -v, тогда в logger.log отобразится **реальный verbose протокол** процесса.
   3. После окончания или падения этого запуска, определить причину неудачи согласно logger.log
   4. Агент **должен** сообщить, что был **повторный запуск**, в чем причина неудачи и спросить, что делать в оба канала:
      - в текущий чат
      - в Telegram
