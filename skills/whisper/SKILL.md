# whisper

## Назначение

Скилл для обработки входящих аудио/голосовых сообщений:
- **вход:** файл в `workspace/media/inbound/<имя_файла>.<ext>`;
- **действие:** перенос файла в `workspace/media/archive/`, расшифровка через fast-whisper, сохранение текста рядом с аудио `<имя_файла>.txt`;
- **выход:** текст расшифровки в `workspace/media/archive/<имя_файла>.txt` и callback-сообщение основному ассистенту.

## Скрипт

Основной скрипт: `%USERPROFILE%/.openclaw/workspace/skills/whisper/transcribe_and_archive.py`

Поведение скрипта:
1. Принимает **один или несколько** позиционных аргументов: **полные пути** к входным аудио файлам (обычно из `workspace_root/media/inbound`).
2. Проверяет, что каждый файл существует.
8. Для **каждого файла** отправляет сервисное сообщение в основную сессию агента через `openclaw agent send`:
   ```
   [transcribe-start] file: <имя_файла>.<ext>
   ```
3. Ищет папку `media` в workspace root и создаёт при необходимости папку `media/archive`.
4. Переносит файл из `путь\к\<имя_файла>.<ext>` в `workspace/media/archive/<имя_файла>.<ext>`.
5. Запускает fast-whisper (через `faster_whisper.WhisperModel`) на файл в `workspace/media/archive/<имя_файла>.<ext>`.
6. Формирует текст:
   - первая строка: `Detected language '<код_языка>' with probability <вероятность>`;
   - далее строки вида `[start -> end] текст сегмента`.
7. Сохраняет результат в `workspace/media/archive/<имя_файла>.txt` (UTF-8).
9. Отправляет в основную сессию агента сообщение через `openclaw agent send`:
   ```
   [transcribe-done] file: <имя_файла>.<ext>
   ```

Всегда запускать **асинхронно** скрипт transcribe_and_archive скила whisper (`path/to/workspace/skills/whisper/transcribe_and_archive.py`).

Скрипт принимает **несколько файлов одной командой** через позиционные аргументы. Если обрабатывается пачка аудио, запускать **одним процессом**, передавая все абсолютные пути подряд. Не запускать один и тот же pipeline самовольно в несколько параллельных процессов.

Базовый запуск:
```bash
python313 <абсолютный_путь_к_workspace>/skills/whisper/transcribe_and_archive.py <абсолютный_путь_к_файлу>
```

Пакетный запуск:
```bash
python313 <абсолютный_путь_к_workspace>/skills/whisper/transcribe_and_archive.py <абсолютный_путь_к_файлу_1> <абсолютный_путь_к_файлу_2> <абсолютный_путь_к_файлу_3>
```

Если входной файл **уже лежит в** `workspace/media/archive/` (или вообще не должен быть перемещён), запускать с флагом **--keep-on-place**:
```bash
python313 <абсолютный_путь_к_workspace>/skills/whisper/transcribe_and_archive.py -k <абсолютный_путь_к_файлу>
```

Если есть подозрение, что обработка **«падает/зависает»** или есть подозрение на зависание, запускать c флагами **--silent --verbose**:
```bash
python313 <абсолютный_путь_к_workspace>/skills/whisper/transcribe_and_archive.py -s -v <абсолютный_путь_к_файлу>
```

Требования:
- запуск через `exec` c `background: true` и `timeout: 600`
- **не** ждать завершения через `process poll`
- скрипту всегда передаётся **абсолютный путь** к исходному файлу
- если файлов несколько, передавать их **в одном запуске** как несколько позиционных аргументов
- не склеивать несколько запусков shell-операторами (`&&`, `;` и т.п.) вместо одного штатного многoфайлового запуска

```bash
python313 <абсолютный_путь_к_workspace>/skills/whisper/transcribe_and_archive.py -s -v <абсолютный_путь_к_файлу>
```

Это эквивалентно ручному запуску распознавания тем же движком без callback-сообщений в main-сессию.

Оценка времени выполнения распознавания: **примерно длина аудиодорожки × 1.17**.

## Поведение ассистента, когда нужно расшифровать аудио файл:

### 1. Ассистент запускает cкрипт всегда с абсолютными путями:

```bash
python313 <абсолютный_путь_к_workspace>/skills/whisper/transcribe_and_archive.py полный/абсолютный/путь/к/<имя_файла>.<ext>
```

- Указывается вегда *полный абсолютный путь* к workspace root!
- Указывается вегда *полный абсолютный путь* к аудио файлу!
- Запуск **всегда асинхронный**, через `exec` с `background: true`!
- **Нельзя ждать завершения процесса** через `process poll`!

Ожидаемая структура путей (относительно workspace):
   - исходный файл лежит в `абсолютный_путь_к_файлу`
После обработки:
   - архивный audio файл в итоге лежит в `path/to/workspace/media/archive/<имя_файла>.<ext>`
   - текстовый файл транскрибации `path/to/workspace/media/archive/<basename>.txt`, где `<basename>` — `<имя_файла>` без расширения.

### 2. При получении сервисного сообщения `[transcribe-start]`
   ```
      [transcribe-start] file: <имя_файла>.<ext>
   ```

   ассистент отвечает **строго одной строкой**:

   ```text
      обработываю <имя_файла>.<ext>
   ```
   Никакого текста дополнительно!

### 3. При получении сервисного сообщения `[transcribe-done]`
   ```
      [transcribe-done] file: <имя_файла>.<ext>
   ``` 
   Основной ассистент выполняет шаги (чтение текста транскрибации, проверка языка, поиск ошибок, категоризация, запись в дневник, идеи, персоналии, команды) **строго** следуя процессу описанному в skill `audio` раздел ### 3.2. `[transcribe-done]`
